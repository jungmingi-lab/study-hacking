# 웹 해킹 학습 로드맵과 기초 개념 정리
## 1. 웹 해킹 분야에서 자주 등장하는 취약점 조사
웹 해킹에서 가장 자주 언급되는 취약점들은 OWASP Top 10을 중심으로 정리할 수 있다. OWASP는 웹 보안 분야의 거대한 몬스터 도감 같은 존재다. 웹 서비스를 공격하거나 방어할 때 가장 먼저 익혀야 할 취약점들을 알려준다.
### 대표적인 웹 취약점
| 취약점 | 설명 | 예시 |
|:-----|:----:|-----:|
| SQL Injection | 입력값을 이용해 데이터베이스 쿼리를 조작하는 공격 |	로그인 우회, 데이터 유출 |
| XSS (Cross Site Scripting) |	악성 스크립트를 삽입하여 다른 사용자의 브라우저에서 실행되게 하는 공격	쿠키 | 탈취, 피싱 |
| CSRF (Cross Site Request Forgery) |	사용자가 모르는 사이에 요청을 보내게 만드는 | 공격	비밀번호 변경, 송금 |
| Broken Access Control	| 권한이 없는 사용자가 다른 사용자의 정보나 관리자 기능에 접근하는 문제 |	URL 변경으로 관리자 페이지 접근 |
| File Upload 취약점 |	악성 파일을 업로드해 서버에서 실행하는 공격 |	웹쉘 업로드 |
| Directory Traversal |	경로 조작을 통해 서버 내부 파일에 접근하는 공격 |	/etc/passwd 열람 |
| Command Injection	| 서버 명령어를 실행하도록 만드는 공격 |	; cat /etc/passwd |
| SSRF	| 서버가 다른 서버로 요청하게 만들어 내부망에 접근하는 공격	| 내부 API 접근 |
| 인증/세션 취약점	| 로그인, 세션 관리가 약해 계정을 탈취할 수 있는 문제 |	세션 탈취 |
| 민감 정보 노출	| 암호화되지 않은 비밀번호, 개인정보가 노출되는 문제	| 평문 저장 |
