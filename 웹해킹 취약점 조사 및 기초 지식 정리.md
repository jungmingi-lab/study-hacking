# 웹 해킹(Web Hacking) 학습 정리

## 1. 미션 목표

웹 해킹(Web Hacking) 분야를 공부하기 위해 필요한 기초 개념과 학습 방향을 파악하고, 대표적인 웹 취약점인 **OWASP Top 10**을 이해하는 것을 목표로 한다.  
또한 실습 가능한 플랫폼을 조사하고, 이를 바탕으로 개인 학습 로드맵을 수립한다.

## 2. 미션 선정 이유

웹 해킹은 정보보안 분야에서 비교적 접근성이 높고, 실제 서비스 환경에서도 자주 문제가 되는 취약점들을 다룬다.  
특히 웹 서비스는 HTTP, 브라우저, 서버, 데이터베이스, 인증/인가 등 여러 기술이 연결되어 있어, 웹 해킹을 학습하면 보안 전반의 기초를 함께 이해할 수 있다.  
따라서 웹 해킹을 공부하기 위해 어떤 개념을 먼저 익혀야 하는지 정리하고, 단계별 학습 계획을 세우는 것은 의미가 크다.

# 3. 웹 해킹에서 자주 등장하는 취약점 조사

## 3-1. OWASP Top 10이란?

OWASP(Open Worldwide Application Security Project)는 웹 애플리케이션에서 가장 중요한 보안 위험을 정리해 발표하는 비영리 프로젝트이다.  
보안 학습 입문자에게는 어떤 취약점이 중요한지 전체 지도를 보여주는 기준 역할을 한다.

현재 공식 최신판은 **OWASP Top 10:2025**이다.

## 3-2. OWASP Top 10:2025 목록

| 순위 | 항목 | 설명 |
|---|---|---|
| A01 | Broken Access Control | 권한이 없는 사용자가 다른 사용자의 데이터나 기능에 접근할 수 있는 문제 |
| A02 | Security Misconfiguration | 서버, 프레임워크, 클라우드, 헤더 설정 등의 잘못된 보안 설정 |
| A03 | Software Supply Chain Failures | 외부 라이브러리, 빌드 환경, 배포 체인 등 공급망에서 발생하는 보안 실패 |
| A04 | Cryptographic Failures | 암호화 미적용, 약한 암호화, 키 관리 실패 등으로 민감 정보가 노출되는 문제 |
| A05 | Injection | SQL Injection, Command Injection, XSS 등 입력값이 코드나 쿼리로 해석되는 문제 |
| A06 | Insecure Design | 설계 단계에서 보안 고려가 부족하여 구조적으로 취약한 상태 |
| A07 | Authentication Failures | 로그인, 세션, 비밀번호 정책 등의 인증 처리 실패 |
| A08 | Software or Data Integrity Failures | 코드나 데이터의 무결성을 검증하지 않아 위변조 위험이 생기는 문제 |
| A09 | Security Logging & Alerting Failures | 침해 시도나 이상 행위를 기록/탐지/대응하지 못하는 문제 |
| A10 | Mishandling of Exceptional Conditions | 예외 상황, 오류 처리, fail-open 등 비정상 상황 처리 미흡 |

## 3-3. 대표 취약점 간단 정리

### 1) SQL Injection
사용자 입력값이 SQL 쿼리에 직접 포함되어 데이터베이스가 의도하지 않은 명령을 수행하는 취약점이다.  
예를 들어 로그인 우회, 데이터 조회, 수정, 삭제가 가능해질 수 있다.

### 2) XSS(Cross-Site Scripting)
공격자가 삽입한 스크립트가 다른 사용자의 브라우저에서 실행되는 취약점이다.  
쿠키 탈취, 피싱, 화면 변조, 사용자 세션 악용 등에 이용될 수 있다.

### 3) CSRF(Cross-Site Request Forgery)
사용자가 로그인된 상태라는 점을 악용하여, 사용자가 의도하지 않은 요청을 보내게 만드는 공격이다.  
예를 들어 비밀번호 변경, 게시글 작성, 송금 요청 등이 발생할 수 있다.

### 4) Broken Access Control
사용자 권한 검증이 부실하여 다른 계정의 정보 조회, 관리자 기능 접근 등이 가능한 취약점이다.  
최근 OWASP에서 가장 중요한 위험으로 계속 강조되고 있다.

### 5) SSRF(Server-Side Request Forgery)
서버가 대신 요청을 보내는 기능을 악용하여 내부망, 메타데이터 서버, 관리자 페이지 등에 접근하게 만드는 취약점이다.

### 6) File Upload / Path Traversal / Command Injection
파일 업로드 검증이 약하거나, 경로 처리 또는 시스템 명령 실행에 사용자 입력이 개입될 때 발생한다.  
웹셸 업로드, 민감 파일 열람, 서버 명령 실행으로 이어질 수 있다.

# 4. 웹 해킹을 공부하기 위해 필요한 기초 지식

## 4-1. HTTP와 웹 구조 이해

웹 해킹을 공부하려면 가장 먼저 **HTTP 요청(Request)과 응답(Response)** 구조를 이해해야 한다.  
브라우저는 서버에 요청을 보내고, 서버는 상태 코드와 함께 HTML, CSS, JS, 이미지 등의 리소스를 응답한다.

### 꼭 알아야 할 개념
- HTTP Method: `GET`, `POST`, `PUT`, `DELETE`
- 상태 코드: `200`, `302`, `403`, `404`, `500`
- 헤더(Header): `Cookie`, `Authorization`, `Content-Type`, `Set-Cookie`
- 쿠키와 세션
- URL, Query String, Body의 차이
- Same Origin Policy, CORS의 기본 개념

## 4-2. 브라우저 동작 방식

브라우저는 HTML을 받아 구조를 해석하고, 필요한 CSS/JS 파일을 추가 요청한다.  
이 과정을 이해해야 XSS, CSRF, 쿠키, DOM 기반 취약점 등을 이해하기 쉽다.

### 알아야 할 포인트
- 브라우저가 HTML/CSS/JS를 어떻게 처리하는지
- 개발자 도구(DevTools) 사용법
- 쿠키/세션 저장 위치
- DOM과 이벤트 처리
- JavaScript가 페이지를 동적으로 바꾸는 방식

## 4-3. 웹 기술 기초

### HTML
페이지 구조를 정의하는 언어이다.  
폼(form), 입력값(input), 링크(a), 스크립트(script) 태그는 취약점 실습에서 자주 등장한다.

### CSS
디자인을 담당하지만, 보안 개념과 직접 연결되지는 않는다.  
다만 브라우저 렌더링 구조 이해에는 도움이 된다.

### JavaScript
클라이언트 측 동작을 담당한다.  
DOM XSS, 토큰 처리, API 호출, 프론트엔드 검증 우회 등을 이해하려면 JavaScript 기초가 필요하다.

### 서버와 데이터베이스
웹 서버와 애플리케이션 서버, DB가 어떤 식으로 연결되는지 알아야 한다.  
예를 들어 사용자가 입력한 값이 서버를 거쳐 DB 쿼리로 전달되는 흐름을 이해하면 SQL Injection을 훨씬 쉽게 이해할 수 있다.

## 4-4. 추가로 익히면 좋은 내용

- 인증(Authentication)과 인가(Authorization)의 차이
- 해시와 암호화의 차이
- REST API 기본 구조
- JSON 형식
- 프록시 도구 사용법(Burp Suite, OWASP ZAP)
- Linux 기본 명령어
- 기본적인 SQL 문법

# 5. 웹 해킹 학습을 위한 실습 플랫폼 조사

## 5-1. PortSwigger Web Security Academy
가장 추천할 만한 입문~중급 학습 플랫폼이다.  
이론 설명과 실습 랩이 함께 제공되며, SQL Injection, XSS, CSRF, SSRF, 인증, 접근제어 등 웹 취약점을 체계적으로 학습할 수 있다.

### 장점
- 무료
- 주제별 설명이 매우 체계적임
- 실습 랩이 실제 공격 흐름과 유사함
- Burp Suite와 연계 학습이 쉬움

### 추천 대상
- 웹 해킹 입문자
- OWASP Top 10을 실습 중심으로 배우고 싶은 학습자

## 5-2. OWASP Juice Shop
의도적으로 취약하게 만든 웹 애플리케이션이다.  
게임처럼 챌린지를 해결하면서 취약점을 찾고 익힐 수 있다.

### 장점
- 설치형 실습 환경
- 초보자용 가이드 제공
- 다양한 취약점이 포함되어 있음
- 점수판 방식이라 학습 동기 부여가 쉬움

## 5-3. OWASP WebGoat
보안 교육용으로 제작된 의도적 취약 애플리케이션이다.  
취약점 설명, 실습, 완화 방법까지 같이 제공하는 점이 강점이다.

### 장점
- “취약점 설명 → 실습 → 대응 방안” 흐름이 좋음
- OWASP Top 10 대부분을 다룸
- 교육 목적에 적합함

## 5-4. DVWA(Damn Vulnerable Web Application)
PHP 기반의 대표적인 웹 취약점 실습 환경이다.  
난이도 조절이 가능하고, 고전적인 웹 취약점 실습에 적합하다.

### 장점
- 구조가 단순해서 입문자가 실습하기 좋음
- SQL Injection, XSS, File Inclusion 등 기초 실습에 적합
- 로컬 환경에서 안전하게 연습 가능

## 5-5. picoCTF
웹 해킹만 전문으로 다루는 플랫폼은 아니지만, 보안 전반을 게임형 문제 풀이로 익히기에 좋다.  
입문자에게 동기부여가 좋고, 웹 문제도 일부 포함된다.

### 장점
- 무료
- 초보자 친화적
- 문제 풀이 방식이라 흥미를 유지하기 쉬움

## 5-6. 실습 시 주의사항
- 반드시 **합법적이고 허용된 환경**에서만 실습해야 한다.
- 로컬, Docker, CTF, 교육 플랫폼 등 승인된 대상만 사용해야 한다.
- 실서비스나 타인의 서버를 대상으로 테스트하면 불법이 될 수 있다.

# 6. 웹 해킹 학습 방향 정리

웹 해킹은 단순히 공격 기법만 외우는 것이 아니라,  
**“웹이 어떻게 동작하는지 이해하고, 그 흐름에서 어디서 검증이 빠지는지 찾는 공부”**에 가깝다.

따라서 학습 순서는 다음처럼 잡는 것이 효율적이다.

## 1단계: 웹 기초 이해
- HTTP 요청/응답
- 쿠키/세션
- HTML, JavaScript 기초
- 클라이언트와 서버 구조
- 데이터베이스와 SQL 기초

## 2단계: 대표 취약점 이해
- SQL Injection
- XSS
- CSRF
- 인증/인가 취약점
- 파일 업로드, 경로 조작, 명령 실행

## 3단계: 도구 사용
- 브라우저 개발자 도구
- Burp Suite Community Edition 또는 OWASP ZAP
- 간단한 실습 환경(DVWA, Juice Shop, WebGoat)

## 4단계: 체계적인 실습
- PortSwigger Academy 랩 풀이
- OWASP 실습 환경 직접 설치
- 취약점 원리와 대응 방안 함께 정리

## 5단계: 기준 문서 읽기
- OWASP Top 10
- OWASP Web Security Testing Guide(WSTG)
- OWASP ASVS

# 8. 느낀 점 / 정리

웹 해킹을 공부하려면 단순히 해킹 기법만 보는 것이 아니라,  
웹 서비스가 어떤 구조로 동작하는지 먼저 이해해야 한다.  
특히 HTTP, 브라우저, HTML, JavaScript, 쿠키/세션, DB 흐름을 이해하면 대부분의 취약점을 훨씬 쉽게 받아들일 수 있다.

또한 OWASP Top 10은 웹 보안 학습의 큰 틀을 잡는 데 매우 유용하다.  
입문 단계에서는 SQL Injection, XSS, CSRF, 인증/인가 문제부터 시작하고, 이후 SSRF, 공급망 보안, 보안 설정 문제 등으로 확장하는 방식이 효율적이다.
